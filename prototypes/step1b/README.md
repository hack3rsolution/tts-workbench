# TTS 클립 작업대 — Step 1B (한자 읽기 검증)

이 단계의 목표:
- **한자 자동 감지** + 가나 매핑이 작동하는지
- **사용자 사전 누적**이 작동하는지
- 가나 치환된 텍스트로 **Gemini TTS가 정확히 발음**하는지

## 구성

- `index.html` — 한자 읽기 통합 검증 앱
- `cloudflare-worker.js` — Step 1과 동일 (이미 배포되어 있으면 재배포 불필요)

## Step 1 통과 후 시작

Step 1에서 Worker 배포가 끝났다면, Worker 코드는 그대로 쓸 수 있어요.
이 단계에서는 `index.html`만 새로 받아서 열면 됩니다.

---

## 사용 흐름

### 1. 설정
- Step 1에서 입력한 Worker URL과 API Key가 자동으로 로드됨 (localStorage 공유)
- 처음이면 동일하게 입력 후 저장

### 2. 대본 + 한자 읽기

기본 대본:
```
約束の場所で待ってる。雨が降っても、僕は行くよ。
```

**한자 분석** 버튼 클릭 → kuromoji가 자동으로 단어 분리 + 가나 추정.

표시 항목:
- **★ 사전**: 사용자 사전에서 자동 적용된 단어 (보라색 배경)
- **추정**: kuromoji가 사전 기반으로 추정한 가나 (회색)
- **미인식**: 가나 추정 실패 — 직접 입력 필요 (주황)
- **편집됨**: 사용자가 수정한 항목 (파란 테두리, 자동으로 사용자 사전에 저장)

각 단어의 가나 입력란을 수정하면:
- 자동으로 **사용자 사전**에 저장됨 (다음 프로젝트에서 자동 로드)
- 하단의 **TTS 전송 텍스트** 미리보기에 즉시 반영

가나 변환되지 않은 한자가 남아있으면 빨간색으로 표시되고 경고가 떠요.

### 3. 음성 변환

목소리 / 모델 / 톤 지시 입력 후 **음성 변환** 클릭.
TTS는 가나 치환된 텍스트로 호출됩니다 (디버그 정보에서 원본/변환 텍스트 둘 다 확인 가능).

### 4. 사용자 사전

- 한 번 입력한 가나는 자동으로 누적
- JSON 내보내기/가져오기로 백업 또는 다른 기기로 이동 가능
- 잘못 저장된 항목은 × 버튼으로 삭제

## 검증 체크리스트

다음이 모두 되면 Step 1B 통과:

- [ ] kuromoji가 4MB 사전을 다운로드하고 "준비 완료" 상태가 됨
- [ ] `約束の場所で待ってる` 분석 시 `約束`, `場所`, `待` 단어가 분리됨
- [ ] kuromoji가 추정한 가나가 합리적임 (`約束`→`やくそく` 등)
- [ ] 가나 수정 시 사용자 사전 영역에 즉시 추가됨
- [ ] 페이지 새로고침 후 다시 분석해도 사전 항목이 ★ 사전으로 자동 채워짐
- [ ] 미리보기에서 한자가 가나로 정확히 치환됨
- [ ] TTS 호출 시 음성이 가나 발음대로 정확히 읽음
- [ ] 디버그 정보에 "원본 텍스트"와 "TTS 전송 텍스트"가 다르게 표시됨

## 알려진 한계

- **kuromoji는 IPADIC 기반**이라 인명·지명·신조어는 미인식 가능 → 사용자가 직접 입력
- **같은 한자가 다른 발음**일 때 (예: `日本` の `にほん/にっぽん`) → 클립별 오버라이드는 Step 2에서 다중 클립 지원 시 본격 도입
- **사용자 사전은 최우선** → kuromoji 추정보다 사전이 항상 우선됨

---

## 문제 발생 시

- **kuromoji 로드 실패**: 인터넷 연결 또는 jsdelivr CDN 차단 환경 확인
- **분석 후 한자가 그대로 남음**: 한자 분석 버튼을 다시 누르거나, 미리보기의 빨간 한자 단어를 직접 사전에 추가
- **가나로 보내도 발음이 이상함**: Gemini가 가끔 가나 텍스트를 어색하게 읽을 수 있음. 필요 시 가나를 더 명확히 표기 (예: `じゅう` 대신 `じゅー`)
